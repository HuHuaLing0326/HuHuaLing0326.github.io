primes

该任务要求实现埃氏筛法得到 2-280 之间的素数。但是实现的要求并不是简单的迭代，而是通过子进程与管道。具体的实现思路为：

  1. 将父进程所有的数写入管道
  2. 创建子进程
  3. 子进程取出管道的第一个数，创建一个新的管道
  4. 取出管道后面的数，如果不是第一个数的倍数，则写入新的管道
  5. 改子进程递归的创建新的子进程，并且传入新的管道

注意此处如果要向管道写入，必须关闭读端，否则会因为数据量太大而无法写入。

find

在 linux 操作系统中，find 命令用于查找指定目录下的指定文件，还可以根据文件的权限、大小、类型、修改时间等多种属性来过滤搜索结果。在 xv6 中，不要求实现后面的功能。本题目可以参照 ls.c 完成。

在 ls.c 里，read(fd, &de, sizeof(de)) == sizeof(de) 表示从打开的文件夹中读取文件夹内的目录和文件。

重要结构体和函数：

  • dirent：代表了目录中的一个项，通常是一个文件或子目录。它包含了文件名、inode 编号、文件类型等信息。在 xv6 中仅包含文件名和 inum 编号 (inum 暂时不知道有什么作用)
  • stat：stat结构体是用于描述文件系统中文件属性的一个重要结构。这个结构体包含了文件的各种元数据，如文件权限、所有者信息、大小、最后访问时间等
  • fstat：它的功能是通过文件描述符来获取文件的状态信息，这些信息包括文件大小、权限、最后访问时间等。

xargs

在 linux 中，xargs 命令可以将标准输入数据转换成命令行参数，实现命令的组合使用。即命令从左往右执行，执行结果作为参数传递给 xargs 后的命令。linux 中的 xargs 更为复杂，xv6 不要求实现更复杂的功能。命令只需要从标准输入中按行(‘\n’)为单位读取该输出内容，并作为参数添加到命令参数。

这种实现方式每读取一行输入就执行一次命令，适用于简单的命令参数传递场景。

根据以上的输出，syscall 显然是由 usertrap 调用的。

Q2：What is the value of p->trapframe->a7 and what does that value represent?

Q2：p->trapframe->a7 的值是多少，该值代表什么？

根据输出的内容，由 num = p->trapframe->a7; 可知 p->trapframe->a7 的值是 0x7。

user/initcode.S 是 xv6 启动的第一个用户程序，其 start 代码段中会将 SYS_exec 送入 a7，然后执行 ecall，切换到内核态，然后操作系统会执行 syscall 函数，将寄存器 a7 的内容送入 num。

查看 kernel/syscall.h，可知输出数字 7 正好对应 SYS_exec 系统调用。

Q3：What was the previous mode that the CPU was in?

Q3：CPU 之前处于什么模式？

根据输出的内容，可知 p /x $sstatus 的状态是 0x200000022。

查看 RISC-V特权指令 文档，搜索 sstatus，发现 sstatus 的第 8 位 SPP，表示是从哪个模式切换过来的；

将 0x200000022 转换为 2 进制，可以看出其 SPP 位为 0，表示从 User-mode 切换来的。

scause 寄存器是一个 SXLEN 位读写寄存器。当陷阱被带入特权模式时，scause被写入一个代码，表示引起陷阱的事件。否则，scause永远不会被实现写入，尽管它可能被软件明确地写入。

Q6：What is the name of the binary that was running when the kernel paniced? What is its process id (pid)?

Q6：当内核崩溃时正在运行的二进制文件的名称是什么？它的进程 ID （pid） 是什么？

根据输出，可以看出程序名为initcode，pid为1。

Xv6 刚启动的时候处于 Machine Mode，完成基本配置工作后就会尽快跳转到 Supervisor Mode，在这个模式下，main 会尽快完成系统的配置工作并创建出第一个用户程序。

initcode 的定义在 user/initcode.S。这个程序就是 Xv6 所创建的第一个用户态程序。initcode 就是利用 exec 这个系统调用来创建出一个更为复杂的可执行程序。它会将 init 这个程序加载到 a0，这个便是需要创建的程序的程序名，而后将参数 argv 加载到 a1。exec 这个系统调用的调用号是 7，所以需要将 7 加载到 a7 上。相关的操作完成后，就会调用 ecall 将控制权交回给操作系统。
