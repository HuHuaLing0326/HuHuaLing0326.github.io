GNN实践二

实践背景：yoochoose电子商务数据集

  • yoochoose-clicks：表示用户的浏览行为，其中session_id表示一次登录，item_id表示浏览的商品。
  • yoochoose-buy：表示用户最终购买的商品，也就是标签

任务要求：将每一个用户的浏览历史构建出一个图，对图进行二分类任务，判断用户最后的浏览的商品是否进行了购买。

将该数据集制作成图神经网络模型能够输入的数据集的样式。

  • 把每一个session_id都当作一个图，每一个图具有多个点和一个标签
  • 每个图中的点就是其item_id，特征暂时用id表示，后续会做embadding

数据集制作流程如下：

  • 首先遍历数据中的每一组session_id，目的是将其制作成(from torch_geometric.data import Data)格式
  • 对每一组session_id的所有item_id进行编码，编码的目的是制作edge_index
  • 转化格式data = Data(x=x, edge_index=edge_index, y=y)
  • 保存数据集到本地

PyG提供两种不同的数据集类：

  1. InMemoryDataset
  2. Dataset

InMemoryDataset类包含如下方法：

  • raw_file_names：它返回一个包含没有处理的数据的名字的list
  • processed_file_names：返回一个包含所有处理过的数据的list。在调用process()这个函数后，通常返回的list只有一个元素，它只保存已经处理过的数据的名字
  • download：这个函数下载数据到你正在工作的目录中，你可以在self.raw_dir中指定。如果你不需要下载数据，你可以在这函数中写一个pass
  • process：这是Dataset中最重要的函数。整合你的数据成一个包含data的list
